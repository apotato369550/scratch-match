● Meat landed. Here's what was added:

  Schema refactor
  - New cv_chunks / job_chunks tables in SQLite hold chunk text.
  - Dropped qdrant_point_ids JSON columns from cvs/jobs — chunk row id IS the
    Qdrant point id (the seam between the two stores).
  - Reset required when pulling this slice: npm run reset.

  Infra
  - docker-compose.yml — Qdrant + Ollama with persistent volumes
    (data/qdrant/, data/ollama/).
  - npm run qdrant:init — creates cv_chunks + job_chunks collections
    (768-dim cosine).
  - npm run setup:models — pulls mistral + nomic-embed-text via Ollama API.

  Library (src/lib/)
  - ollama.ts — embed / chat / ping / listModels / pullModel; verbose logs.
  - qdrant.ts — thin client + collection helpers; vectors-only payload
    invariant (cv_id, seeker_id, specialization for cv_chunks; job_id,
    employer_id, status, location for job_chunks — never text).
  - chunk.ts — ~500-token, 50-overlap word-based splitter.
  - pdf.ts — pdf-parse wrapper with PdfTooSparseError for scanned PDFs.
  - ingest.ts — shared ingestCv / ingestJob / deleteCv / deleteJob pipeline
    used by both API routes and scripts.
  - match.ts — top-20 per chunk → mean of top-3 chunk scores per job.

  API routes
  - POST/GET /api/seeker/cvs (multipart PDF or text + specialization).
  - DELETE /api/seeker/cvs/:id (cleans up Qdrant points first).
  - GET /api/seeker/matches/:cvId (ranked jobs with similarity scores).
  - POST/GET /api/employer/jobs.
  - All routes role + ownership checked; quotas enforced
    (5 CVs/specialization, 5 open jobs/employer).

  UI
  - /seeker/cvs/upload form is now a real client component that POSTs to
    /api/seeker/cvs and redirects to /seeker/matches?cv=<id> on success.

  Tests
  - scripts/test-bones.ts → scripts/tests/db-auth.ts (renamed).
  - scripts/tests/rag.ts — full round-trip:
      signup seeker + employer
      → CV ingest via /api/seeker/cvs
      → assert SQLite cv_chunks count == Qdrant point count for that CV
      → assert Qdrant payload has no `text` field (vectors-only invariant)
      → post relevant + irrelevant jobs
      → /api/seeker/matches/:cvId — assert relevant outranks irrelevant
      → employer cannot read seeker matches (403)
      → cleanup
  - npm run test runs both db-auth + rag.

  Other
  - ingest-sample.ts now actually chunks + embeds (no more relational-only).
  - package.json adds @qdrant/js-client-rest, pdf-parse.
  - src/types/shims.d.ts declares the pdf-parse internal entrypoint.

  To run end-to-end (your install):
  docker compose up -d
  npm install
  npm run reset
  npm run db:migrate && npm run seed:admin
  npm run setup:models       # slowest step, ~4 GB first-time download
  npm run qdrant:init
  npm run ingest:sample
  npm run dev                # in one terminal
  npm run test               # in another
